When HTML5 has become a standard? What were the main reasonsfordeveloping new standard?
On 28 October 2014, HTML5 was released as a W3C Recommendation, bringing the specification process to completion. Its objectives were to improve the language with support for new multimedia and other new features to keep the language both easily readable by humans and consistently understood by computers and devices such as web browsers, parsers, and so on, as well as to be backward compatible with existing web browsers.

What is a difference between W3C HTML5 and WHATWG HTML Living Standard?
The W3C "HTML5" standard was snapshot-based (HTML5, HTML 5.1, etc.) and static, while the WHATWG "HTML living standard" is continually updated. The relationship had been described as "fragile", even a "rift", and characterized by "squabbling".

What is the minimal HTML page code?
The minimal HTML page code defines the basic structure of an HTML document, including elements like DOCTYPE html, html, head, meta, title, and body, providing essential information and content for rendering a webpage in a browser.

Comment on semantic and presentational markup. Why presentational markup is considered deprecated now?
Semantic markup focuses on using HTML elements to convey the meaning and structure of the content, making it more understandable for both humans and machines. It emphasizes the purpose of the content rather than its appearance.
Presentational markup, on the other hand, involves using HTML elements primarily for styling and visual presentation purposes. It focuses on how the content looks rather than its underlying meaning or structure.
Presentational markup is considered deprecated because it mixes content with styling, making it harder to maintain, less accessible, and less adaptable to different devices and platforms. Separating content and presentation using CSS allows for cleaner code, better accessibility, and easier updates.

What are the audio and video formats, officially supported by HTML standard? What happens if you use other format?
The HTML standard officially supports specific audio and video formats such as MP3, Ogg Vorbis, WAV for audio, and MP4 (H.264), WebM (VP8 or VP9) for video. Using unsupported formats can lead to varying behavior in different browsers, potentially resulting in playback issues or requiring users to download the file instead of playing it directly. To ensure consistent playback across browsers and devices, it's advisable to stick to the officially supported formats.
