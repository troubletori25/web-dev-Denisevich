What is selector's specificity and how it's calculated?

Specificity is the means by which browsers decide which CSS property values are the most relevant to an element and, therefore, will be applied. Specificity is based on the matching rules which are composed of different sorts of CSS selectors.

An explanation of how specificity is calculated:

Inline Styles: Styles applied directly to an HTML element using the style attribute have the highest specificity.

ID Selectors: Selectors targeting elements by their ID (e.g., #example) have a higher specificity than other types of selectors.

Class Selectors, Attribute Selectors, and Pseudo-classes: Selectors targeting elements by class, attribute, or pseudo-classes (e.g., .example, [type="text"], :hover) have a medium level of specificity.

Type Selectors and Pseudo-elements: Selectors targeting elements by their tag name (e.g., div, p) or pseudo-elements (e.g., ::before, ::after) have the lowest specificity.

Start at 0, add 100 for each ID value, add 10 for each class value (or pseudo-class or attribute selector), add 1 for each element selector or pseudo-element.


What are complex selectors?

Complex selectors are like combinations of different instructions in CSS that allow you to target elements in more specific ways. Instead of just using one type of instruction, like targeting elements by their class or ID, complex selectors let you mix and match different types of instructions to target elements based on their relationships with other elements, their attributes, or their positions in the HTML structure. It's like using a combination of clues to find exactly the element you're looking for on a webpage.


What is the difference between pseudo-classes and pseudo-elements?

Pseudo-classes are like extra instructions you add to target elements based on their state or interaction, like when a user hovers over them or when they're the first child of their parent. They're like adding conditions to your CSS rules.

Pseudo-elements, on the other hand, are like creating imaginary elements within real ones, allowing you to style specific parts of an element's content, like the first letter or the first line of a paragraph. They're like adding extra elements to your HTML without actually changing the HTML structure.

To calculate specificity, assign each type of selector a value:

Inline styles: 1,0,0,0

ID selectors: 0,1,0,0

Class selectors, attribute selectors, and pseudo-classes: 0,0,1,0

Type selectors and pseudo-elements: 0,0,0,1
